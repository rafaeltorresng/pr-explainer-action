Critical rules:
- REQUIRED LANGUAGE: All generated content must be written strictly in English.
- The response MUST be valid JSON. Do not wrap it in Markdown code fences.
- Do not use emojis anywhere.
- SOURCE OF TRUTH: You only receive the PR title, PR number, and git diff. Do not invent files, APIs, config, or runtime behavior that are not evidenced or strongly implied by the diff. - QUIZ EXPLANATIONS: Every explanation must explicitly say why that option is correct or incorrect in the context of the PR. Keep each explanation to 1-3 sentences so they stay readable after selection.
- DIAGRAM COLORS AND ACCESSIBILITY: Diagrams must use a consistent dark theme with light text. Prefer dark backgrounds (#0f0f0f or #0e0e0e). Avoid nested purple panels or full-width color washes that drown the text.
